    By looking at it we can see that it’s encoded hex, and something else. Decode it using https://malwaredecoder.com/ javascript:(function(){ document“getElementById”[“value”]= “clave”; document“getElementById”[“value”]= “11des0081x”“;

    Our first set of creds - clave:11des0081x

  3. The creds don’t work for SSH, but they do work on the login-page on port 80. Looking around on the page we find a few things .. http://10.10.10.114/root/profile: “TODO: Connect with postgresql” .. http://10.10.10.114/snippets/1: <?php $db_connection = pg_connect(“host=localhost dbname=profiles user=profiles password=profiles”); \(result = pg_query(\)db_connection, “SELECT * FROM profiles”);

    psql is not reachable from our remote machine, maybe this is our way towards user / root from init.

  4. We are able to upload files in http://10.10.10.114/root/profile/tree/master, upload a shell (webshell or reverse php) Using dirb we can locate the url for our shell. We know that index.php, README.md and developer.jpg are in the same folder.
